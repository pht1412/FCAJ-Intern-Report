This section lists and introduces the technical blogs authored and published to the [AWS Study Group VN](https://www.facebook.com/groups/awsstudygroupfcj) community. These posts summarize the practical experiences, architectural discussions, and latest cloud technology updates gathered during the project implementation.

### [Blog 1: Architectural Discussion of the Mini Social Network Project](3.1-Blog1/)
This blog details the architectural design of the "Mini Social Network" project using the AWS 3-Tier Architecture model. It logically breaks down the system into the Data Plane (User Access & Data Processing) and Control Plane (Operations & CI/CD), highlighting the team's approach to infrastructure isolation and security. It also features a valuable Q&A section based on practical feedback from senior AWS community members.

### [Blog 2: Cloud Deployment Journey – From a $35 Lesson to a Cost-Optimized Architecture](3.2-Blog2/)
This post shares a hands-on journey of migrating a monolithic application from a traditional VPS to a Cloud-Native architecture on AWS. It openly discusses overcoming practical challenges such as unexpected NAT Gateway costs, SPA routing errors, and performance bottlenecks. The blog showcases how the team divided tasks by specialized roles (Ops, UI/UX, Observability, DevSecOps) to achieve a highly optimized and secure infrastructure.

### [Blog 3: Amazon ECS Introduces High-Resolution Metrics for 4x Faster Service Auto Scaling](3.3-Blog3/)
Based on the latest updates from AWS, this analytical article explores the newly introduced high-resolution (20-second interval) metrics for Amazon ECS. It explains the core mechanism of how this breakthrough reduces scale-out delays by up to 76% (from 363s down to 86s), enabling cloud systems to respond instantly to traffic spikes while simultaneously reducing preemptive capacity padding and compute costs.
