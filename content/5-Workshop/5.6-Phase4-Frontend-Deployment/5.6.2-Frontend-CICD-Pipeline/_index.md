## Objective of this Lab

In this lab, you will configure Jenkins to automate the deployment of the MiniSocial Frontend.

You will create the AWS credentials required by the frontend pipeline, update the Jenkinsfile using the infrastructure outputs generated in the previous lab, configure a Jenkins Pipeline Job, and execute a complete CI/CD workflow that builds the React application, uploads it to Amazon S3, and invalidates the Amazon CloudFront cache automatically.

---

## Learning Objectives

After completing this lab, you will be able to:

- Configure AWS Credentials for the Frontend Jenkins Pipeline
- Update the Frontend Jenkinsfile using your CloudFormation Outputs
- Configure a Jenkins Pipeline Job for the React application
- Automatically build the frontend using Jenkins
- Deploy the generated static website to Amazon S3
- Automatically invalidate the Amazon CloudFront cache after every deployment

# Step 1 – Create AWS Credentials for the Frontend Pipeline

Although AWS Credentials have already been configured for the Backend Pipeline, the **frontend/Jenkinsfile** uses a different Credentials ID.

Navigate to:

```text
Manage Jenkins
    └── Credentials
            └── System
                    └── Global Credentials
```

Click:

```text
Add Credentials
```

Configure the credential using the following values.

| Field | Value |
|-------|-------|
| Kind | AWS Credentials |
| Access Key ID | Your AWS Access Key |
| Secret Access Key | Your AWS Secret Key |
| ID | aws-frontend-creds |
| Description | AWS Credentials for Frontend Pipeline |

Click:

```text
Create
```

---
**Credential Recommendation**

{{% notice tip %}}

You may reuse the same IAM User Access Key that was configured for the Backend Pipeline.
Ensure that the Jenkins Credential ID is exactly:
`aws-frontend-creds`
This Credential ID is referenced directly by **frontend/Jenkinsfile**. If the ID does not match, Jenkins will not be able to authenticate with AWS during the deployment.

{{% /notice %}}

# Step 2 – Update the Frontend Jenkinsfile

The provided **frontend/Jenkinsfile** contains example values for the Amazon S3 Bucket and CloudFront Distribution.

Before running the pipeline, replace these values with the outputs generated by the Frontend CloudFormation Stack in **Lab 5.6.1**.

Open:

```text
frontend/Jenkinsfile
```

Locate the following **environment** block.

```groovy
environment {
    AWS_REGION         = "us-east-1"

    S3_BUCKET_NAME     = "minisocial-frontend-xxxxxxxx"

    CLOUDFRONT_DIST_ID = "XXXXXXXXXXXX"

    VITE_API_BASE_URL  = "https://api.minisocial-network.id.vn"
}
```

Update the following variables.

| Variable | Replace With |
|----------|--------------|
| AWS_REGION | us-east-1 |
| S3_BUCKET_NAME | FrontendBucketName from CloudFormation Outputs |
| CLOUDFRONT_DIST_ID | CloudFrontDistributionId from CloudFormation Outputs |
| VITE_API_BASE_URL | Your Backend API Domain |

---
**Before continuing, verify the following:**

{{% notice warning %}}

 `AWS_REGION = "us-east-1"` is configured correctly.
 Your ACM certificate was issued in **US East (N. Virginia) (`us-east-1`)**.
 Your Frontend infrastructure is deployed in **`us-east-1`**.
**Important:** Amazon CloudFront only supports ACM certificates issued in **US East (N. Virginia)**. Deploying the Frontend stack in another Region may prevent HTTPS from working correctly.

{{% /notice %}}

---

After updating the Jenkinsfile:

- Commit the changes.
- Push the updated code to your GitHub repository.

# Step 3 – Configure the Frontend Pipeline Job

Return to the Jenkins Dashboard.

Click:

```text
New Item
```

Create a new **Pipeline** project.

For example:

```text
MiniSocial-Frontend-Pipeline
```

---

Under the **Pipeline** section, configure the following settings:

| Setting | Value |
|---------|-------|
| Definition | Pipeline script from SCM |
| SCM | Git |
| Repository URL | Your GitHub Repository URL |
| Credentials | Your GitHub Credentials |
| Branch | main (or your working branch) |
| Script Path | frontend/Jenkinsfile |

Configure the Pipeline as shown in the image below.

![Configure Jenkins Pipeline](/images/5-Workshop/5.5-Phase3-Backend-Deployment/ConfigPipeline.png)
<center><i>Configure the Pipeline so Jenkins fetches the Jenkinsfile directly from GitHub using the Pipeline script from SCM option.</i></center>

After completing the configuration, click:

```text
Apply
```

Then click:

```text
Save
```

---

{{% notice tip %}}
Using **Pipeline script from SCM** ensures Jenkins always executes the latest version of the Frontend Jenkinsfile stored in your GitHub repository.
{{% /notice %}}

---

Return to the main Jenkins Dashboard. You should see the newly created Pipeline ready for execution.

![Jenkins Dashboard](/images/5-Workshop/5.6-Phase4-Frontend-Deployment/DashboardJenkins.png)
<center><i>The Jenkins Dashboard displaying the successfully created Pipeline.</i></center>

# Step 4 – Execute the Frontend Pipeline

Before running the pipeline, verify the following:

- Jenkins is running.
- Docker Desktop is running.
- Internet connectivity is available.
- The latest Jenkinsfile has been pushed to GitHub.

---
### Important Region Verification

{{% notice warning %}}

**Region Verification**
Before running the pipeline, make sure that:
`AWS_REGION = "us-east-1"` is configured in **frontend/Jenkinsfile**.
The Amazon S3 bucket and Amazon CloudFront distribution were both deployed in **US East (N. Virginia) (`us-east-1`)**.
Using a different Region may cause the deployment to upload files to the wrong Amazon S3 bucket or prevent Jenkins from updating the correct CloudFront distribution.

{{% /notice %}}

---

Click:

```text
Build Now
```

Monitor the Console Output.

If everything is configured correctly, Jenkins will automatically perform the following stages:

1. Clone the GitHub repository.
2. Install Node.js dependencies.
3. Build the React (Vite) application.
4. Upload the generated **dist/** folder to Amazon S3.
5. Remove outdated files from the S3 bucket.
6. Create a CloudFront cache invalidation (`/*`).

Wait until the Pipeline finishes successfully.

The completed pipeline should look similar to the following example.

![Successful Frontend Pipeline](/images/5-Workshop/5.6-Phase4-Frontend-Deployment/BuildSuccess_FE.png)

<center><i>The Frontend Pipeline successfully builds the React application, uploads the generated files to Amazon S3, and invalidates the Amazon CloudFront cache, making the latest website immediately available to users.</i></center>
